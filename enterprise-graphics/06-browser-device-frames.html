1. Browser Frame Anatomy

A complete browser frame has 4 layers from top to bottom: Tab bar (tabs with one active), Toolbar (dots + nav arrows + URL bar + action buttons), Separator (1px border-bottom), Viewport (the content). The tab bar and toolbar use slightly different background shades (#2a2724 vs #222019) to create visual separation without harsh lines. The active tab matches the toolbar background color, making them appear connected.

2. URL Bar Details

The lock icon is built with a div (the body) and a ::before pseudo-element (the shackle). Green color (#28c840) signals HTTPS. The URL text splits the domain into a brighter color and the path into a dimmer one. This is exactly how Chrome renders URLs. Use text-overflow: ellipsis and white-space: nowrap to handle long URLs gracefully.

3. Phone Frame Anatomy

The outer frame uses a large border-radius: 40px with thick padding (12px) to create the device bezel. The screen inside has a slightly smaller radius (border-radius: 30px). The Dynamic Island is a centered pill shape (border-radius: 20px). The status bar shows time on the left, signal/battery on the right. The home indicator is a centered 120px wide bar at the bottom. All of these are pure CSS -- no images needed.

4. Laptop Frame Anatomy

The screen is the main container with rounded top corners and no bottom radius. The camera dot is centered at the top. The keyboard base extends slightly wider than the screen (width: 108%; margin-left: -4%) to create the realistic perspective of a laptop's base being wider than its lid. A ::before pseudo-element creates the trackpad notch. The base uses a subtle gradient from top to bottom for depth.

5. Making Frames Responsive

Browser frames use max-width and scale down naturally. Phone frames use a fixed width (280px is standard for marketing graphics) and can be centered. Laptop frames also use max-width. The key is that all internal elements use relative sizing (percentages, em, or rem) so they scale with the frame. Never use fixed pixel sizes for content inside frames.

6. Enterprise Polish

The difference between amateur and enterprise frames comes down to three details: (1) Layered shadows on the outer frame for realistic depth; (2) Inner border on the phone frame (inset 0 0 0 2px) to simulate the screen edge; (3) Consistent color temperature -- the frame chrome should be warm-gray to match the dark theme, not cold-gray. Webflow and Framer both add a subtle glow behind their device frames using a radial gradient on the parent container.
